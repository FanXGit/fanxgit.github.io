# 1.Linux 设置开机启动脚本

# 1.1.设置nacos 开机启动流程

1、创建 Nacos 服务的单元文件：
使用任何文本编辑器，创建一个名为 nacos.service 的文件，并将以下内容粘贴到该文件中：

[Unit]
Description=Nacos Server
After=network.target

[Service]
Type=forking
ExecStart=/usr/local/src/nacos/nacos_start.sh
ExecReload=/usr/local/nacos/bin/shutdown.sh
ExecStop=/usr/local/src/nacos/nacos_stop.sh
PrivateTmp=true
# Type=forking
#      ExecStart=/usr/local/nacos/bin/startup.sh -m standalone
#      ExecReload=/usr/local/nacos/bin/shutdown.sh
#      ExecStop=/usr/local/nacos/bin/shutdown.sh
#      PrivateTmp=true

[Install]
WantedBy=multi-user.target

3、设置文件权限： 运行以下命令，以确保单元文件的权限正确设置：

sudo chmod 644 /etc/systemd/system/nacos.service
1

4、重新加载 systemd 守护程序： 运行以下命令，重新加载 systemd 守护程序以使其注意到新的单元文件：

sudo systemctl daemon-reload
1

5、启用 Nacos 服务： 运行以下命令，以启用 Nacos 服务并设置其在系统启动时自动启动：

sudo systemctl enable nacos
1

6、现在，当您的系统启动时，Nacos 服务将自动启动。您可以使用以下命令来控制 Nacos 服务的启动、停止和状态：

sudo systemctl start nacos  # 启动 Nacos 服务
sudo systemctl stop nacos   # 停止 Nacos 服务
sudo systemctl status nacos # 查看 Nacos 服务的状态

# 2、scp 常用命令

在Linux系统中，scp命令用于在本地和远程系统之间进行安全的文件传输。它基于SSH协议，支持加密和身份验证。下面是一些常用的scp命令参数：

# 2.1、常用参数

常用参数 含义
-r 递归地复制目录及其内容
-P <port> 指定远程SSH服务器的端口号
-p 保留源文件的修改时间和权限
-q 静默模式，不显示传输过程信息
-C 开启压缩，加快传输速度
-i <identity_file>：使用指定的私钥文件进行身份验证
-l <limit>：限制带宽，控制传输速度
-B 使用批处理模式，将多个文件一次性传输
-v 显示详细的调试信息

# 2.2、实用场景

# 1、从本地复制文件到远程服务器:

scp local_file remote_user@remote_host:remote_folder/ #这个命令将本地的local_file文件复制到远程服务器的remote_folder/目录下。
#举例:
scp README.MD root@123.249.77.134:/usr/local/ # 将本地README.MD 复制到123.249.77.134服务器 /usr/local/ 下。 注:服务器没有/local文件夹会报错
1
2
3

# 2、从远程服务器递归复制整个目录到本地

scp -r remote_user@remote_host:remote_folder/ local_folder/  #这个命令将远程服务器上的remote_folder/目录及其内容递归地复制到本地的local_folder/目录下。
1
2

请注意，scp 命令执行时会提示您输入远程服务器的密码，除非您已经配置了 SSH 密钥认证。

# 2.1、本地命令连接

本地shell 安装ssh 命令，使用ssh命令远程连接服务器
-p 指定连接的linux的端口号

ssh  user@hostname -p 2222

# SSH 常用的命令

在Linux系统中，ssh命令用于建立与远程服务器之间的安全Shell会话。它基于SSH协议，提供加密的远程登录和执行命令的功能。下面是一些常用的ssh命令参数：

# 常用参数

参数 作用
-p <port> 显示所有活动的连接和监听端口。
-i<identity_file> 使用指定的私钥文件进行身份验证
-l<username> 指定远程登录的用户名。
-X 启用X11转发，支持远程图形界面应用程序。
-A 开启身份验证代理，允许通过跳板机（Jump Host）登录其他服务器。
-t 强制为远程会话分配伪终端，通常用于执行需要交互的命令。
-C 开启压缩，加快数据传输速度。
-q 静默模式，减少输出信息。
-v 显示详细的调试信息。
-L <local_port>:<target_host>:<target_port> 本地端口转发，将本地端口与远程目标主机和端口进行关联。
-R <remote_port>:<target_host>:<target_port> 远程端口转发，将远程端口与本地目标主机和端口进行关联。

# 实用场景

# 1、建立与远程服务器的SSH会话:

ssh <username>@<remote_host> #通过这个命令，你可以使用指定的用户名<username>和远程服务器的地址<remote_host>建立SSH连接。
1

# 2、建立通过跳板机登录其他服务器的SSH会话:

ssh -A <jump_host> ssh <username>@<target_host> #这个命令使用跳板机（Jump Host）登录到其他服务器。你需要指定跳板机的地址<jump_host>、目标服务器的用户名<username>和目标服务器的地址<target_host>。
1

# 3、使用本地端口转发:

ssh -L <local_port>:<target_host>:<target_port> <username>@<remote_host>#这个命令将本地端口<local_port>与远程目标主机<target_host>和端口<target_port>进行关联。
1
2

# 4、使用远程端口转发:

ssh -R <remote_port>:<target_host>:<target_port> <username>@<remote_host> #这个命令将远程端口<remote_port>与本地目标主机<target_host>和端口<target_port>进行关联。
1

# 3、netstat 常用命令

Netstat（网络统计）是一个用于检查网络连接和网络统计信息的命令行工具。它在多个操作系统上都可用，如Linux、Windows等。Netstat 提供了关于网络连接、网络接口、路由表和网络统计信息的详细信息，能够帮助管理员进行网络故障排查和性能分析。

# 3.1、常用参数

参数 作用
-a（--all） 显示所有活动的连接和监听端口。
-n（--numeric） 以数字形式显示地址和端口号，而不进行名称解析。
-p（--program） 显示与每个连接关联的程序/进程的名称和进程ID。
-t（--tcp） 显示TCP连接信息。
-u（--udp） 显示UDP连接信息。。
-l（--listening） 显示路由表信息。
-s（--statistics） 显示各个协议的统计信息。
-e（--extend） 显示扩展的详细信息，如用户和组ID等。

# 3.2、实用场景

# 1、查看正在运行的端口号:

1、

netstat -tunl  #命令会显示系统上所有的 TCP 和 UDP 连接信息，包括正在监听的端口，
               #并且会显示与每个连接关联的程序/进程的名称和进程ID。
              #该命令对于查看当前系统上的网络连接状态和监听端口以及它们所属的程序非常有用
1

# 4、查看试试服务占用情况

1、top命令：它会显示当前运行的进程和它们的资源使用情况。在终端中输入以下命令：

top
1

top命令将会实时地显示进程的资源占用情况，包括CPU利用率、内存使用量、进程ID等。

2、htop命令：它是一个交互式的进程查看器，提供了更友好和直观的界面以查看进程的占用情况。如果没有安装htop，你可以使用适合你的Linux发行版的包管理器进行安装。例如，对于Ubuntu或Debian系统，可以使用以下命令进行安装：

sudo apt-get install htop
1

安装完成后，在终端中输入以下命令启动htop：

htop
1

htop将会显示类似于top命令的进程信息，但以更易读和交互的方式展示。

3、ps命令：它用于列出当前运行的进程。可以结合不同的选项使用ps命令 ，例如：

ps -ef  # 列出所有进程：
ps -u username  #列出特定用户的进程
ps -p PID -o %cpu,%mem,cmd #查找特定进程的详细信息 将PID替换为要查找的进程ID。
1
2
3
4

# 看每一个进程的内存使用情况

ps aux --sort=-%mem #每一个进程的内存使用情况

# 5、ls 常用命令

ls命令用于列出当前目录中的文件和子目录。ls命令有很多参数，下面是一些常用的ls命令参数：

# 5.1、常用参数

参数 作用
-l 以长格式显示文件和目录的详细信息，包括权限、所有者、文件大小等。
-a 显示所有文件和目录，包括以"."开头的隐藏文件
-h 以人类可读的格式显示文件和目录的大小
-r 反向排序，按照降序显示文件和目录
-t 按照修改时间排序，最新修改的文件和目录显示在前面
-S 按照文件大小排序，文件大小大的显示在前面
-R 递归地列出所有子目录的文件和目录
-d 仅显示目录本身，而不显示其下的文件和目录
-i 显示文件和目录的inode号
--color 以彩色显示文件和目录

# 5.2、实用场景

# 1、查看所有文件夹的大小（包括隐藏文件）

ls -lha #以长格式显示所有文件和目录的详细信息，包括以"."开头的隐藏文件，以人类可读的格式显示文件大小。这个参数组合常用于查看整个目录结构的详细信息。
1
2

# 2、查看所有文件夹按照最新修改的时间排序

ls -lt #按照修改时间排序，最新修改的文件和目录显示在前面。这个参数组合适用于查找最近修改的文件或目录。
1

# 3、按照文件夹的大小排序 查看所有文件夹

ls -lSrh #以长格式显示文件和目录的详细信息，按照文件大小排序，文件大小大的显示在前面，以人类可读的格式显示文件大小，并以逆序方式显示。这个参数组合常用于查找占用空间最大的文件。
1
2

# 4、查看文件下所有的目录与文件

ls -Rl #以长格式递归地列出所有子目录的文件和目录的详细信息。这个参数组合适用于查看整个目录树的详细信息。
1
2

# 5、查看目录自身信息，不显示目录详细内容

ls -ld #仅显示目录本身，而不显示其下的文件和目录。这个参数组合常用于只关注目录本身而不需要查看目录内的内容。
1

# 6、显示文件和目录的inode号

ls -i #显示文件和目录的inode号。这个参数组合可以用于查找具有相同inode号的文件，以确定是否为硬链接文件。
1

# 7、查看当前目录下每个文件夹所占的空间

du -shc */ | sort -h
1
