Java 各版本主要新特性总结

机翻自 azatsatklichov/Java-Features (opens new window)，待校正。

其中包含在该版本中预览、实验性、孵化中的功能，使用前请先查阅官方文档。

# Java 1（1996-01 发布），Java 1.1（1997-02 发布）

Java 1 版本于 1996 年 1 月 23 日发布。首个稳定版本 JDK 1.0.2 被称为 Java 1。Java 1.1 版本于 1997 年 2 月 19 日发布，其主要增加了以下内容：

  • 对 AWT 事件模型进行了广泛的改进
  • 内部类被添加到语言中
  • JavaBeans
  • JDBC
  • RMI
  • 反射机制仅支持内省，无法在运行时修改（在 J2SE 1.2 中引入了 AccessibleObject 类及其子类（例如 Field 类），从而使得可以通过反射机制对对象进行修改）
  • 在 Microsoft Windows 平台上，由 Symantec 为 JavaSoft 开发的 JIT（Just In Time）编译器
  • 国际化和来自 Taligent 的 Unicode 支持

# Java 2（1998-12 发布）

代号为 Playground。1998 年 12 月 8 日发布的 Java 2 版本及其后续发布被追溯地重新命名为 Java 2，版本名称 "J2SE"（Java 2 Platform, Standard Edition）取代了 JDK，以区分基础平台与 J2EE（Java 2 Platform, Enterprise Edition）和 J2ME（Java 2 Platform, Micro Edition）。

这是 Java 的一个非常重要的版本，它将 Java 平台的规模扩大了三倍，达到了 59 个包含 1520 个类的规模。主要新增功能包括：

  • strictfp 关键字
  • Swing 图形 API 被整合到核心类中
  • Sun 的 JVM 首次配备了 JIT（即时编译器）
  • Java 插件
  • Java IDL，用于 CORBA 互操作性的 IDL 实现
  • 集合框架（Collections framework）

# Java 3（2000-05 发布）

代号为 Kestrel，详细的 API 文档可以在 此处 (opens new window) 找到。2000 年 5 月 8 日发布的版本带来了以下最显著的变化：

  • 包含 HotSpot JVM（HotSpot JVM 最早在 1999 年 4 月为 J2SE 1.2 JVM 发布）
  • RMI 被修改以支持与 CORBA 的可选兼容性
  • Java 命名和目录接口（Java Naming and Directory Interface，JNDI）被包含在核心库中（之前只作为扩展可用）
  • Java 平台调试器架构（Java Platform Debugger Architecture，JPDA）
  • JavaSound
  • 合成代理类（Synthetic proxy classes）

# Java 4（2002-02 发布）

代号为 Merlin，详细的 API 文档可以在 此处 (opens new window) 找到。2002 年 2 月 6 日发布的版本是第一个在 Java 社区进程（Java Community Process）下开发的 Java 平台版本，编号为 JSR 59。该版本的主要变化包括：

  • 语言变更
  • assert 关键字（JSR 41 中指定）
  • 库改进
  • 模仿 Perl 正则表达式的正则表达式
  • 异常链，允许异常封装原始的低级异常
  • 支持 Internet 协议第 6 版（IPv6）
  • 非阻塞 I/O（Java 中称为 NIO）（在 JSR 51 中指定）
  • 日志记录 API（在 JSR 47 中指定）
  • 图像 I/O API，用于读取和写入 JPEG 和 PNG 等格式的图像
  • 集成的 XML 解析器和 XSLT 处理器（JAXP）（在 JSR 5 和 JSR 63 中指定）
  • 集成的安全性和加密扩展（JCE、JSSE、JAAS）
  • 包含了 Java Web Start（Java Web Start 首次在 2001 年 3 月为 J2SE 1.3 发布）（在 JSR 56 中指定）
  • 偏好设置 API（java.util.prefs）

Java 1.4 的公共支持和安全更新于 2008 年 10 月结束。Oracle 客户的付费安全更新于 2013 年 2 月结束。

# Java 5（2004-09）

Java SE 5 文档 (opens new window)，API 文档 (opens new window)，特性 (opens new window)

代号为 Tiger，于 2004 年 9 月 30 日发布，最初的版本号为 1.5，该版本号仍然作为内部版本号使用。这个版本的编号被改为 "更好地反映 J2SE 的成熟度、稳定性、可扩展性和安全性"。此版本是在 JSR 176 下开发的。J2SE 5.0 在 2008 年 4 月 8 日进入了公共更新期的结束阶段；自 2009 年 11 月 3 日起，公众无法获取更新。有限的更新提供给付费的 Oracle 客户，直到 2015 年 5 月为止。Tiger 添加了许多重要的新语言特性：

  • 泛型：为集合提供编译时（静态）类型安全，并消除了大多数类型转换的需求（由 JSR 14 指定）
  • 元数据：也称为注解（annotations），允许在类和方法等语言结构上添加额外的数据，这些数据可以被元数据感知的工具进行处理（由 JSR 175 指定）
  • 自动装箱/拆箱：原始类型（如 int）和原始包装类（如 Integer）之间的自动转换（由 JSR 201 指定）
  • 枚举：enum 关键字创建了一个类型安全的有序值列表，以前只能通过非类型安全的常量整数或手动构建的类来实现（通过 JSR 201 指定）
  • 可变参数：现在可以使用类型名称后跟三个点（例如 void drawText(String... lines)）来声明方法的最后一个参数，在调用代码中可以使用任意数量的该类型参数，并将它们放入数组中传递给方法，或者调用代码可以传递一个该类型的数组
  • 增强的 for 循环：扩展了 for 循环语法，可以用于迭代数组或任何 Iterable 的成员，例如标准的 Collection 类（由 JSR 201 指定）
  • 改进的多线程 Java 程序的执行语义：新的 Java 内存模型解决了以前规范中的复杂性、有效性和性能问题
  • 静态导入

此外，标准库还进行了以下改进：

  • 为 RMI 对象自动生成存根（stub）
  • Swing：新增了可定制外观和皮肤，称为 synth
  • java.util.concurrent 包中的并发工具
  • Scanner 类用于解析来自各种输入流和缓冲区的数据

版本号 1.5.0 还是 5.0？这种对应关系延续到后续版本（Java 6 = JDK 1.6，Java 7 = JDK 1.7，以此类推）。

# Java 6（2006-12 发布）

Java SE 6 文档 (opens new window)，API 文档 (opens new window)，Open JDK (opens new window)

代号为 Mustang，从 2006 年 12 月 11 日发布的版本开始，Sun 用 Java SE 取代了 "J2SE" 的名称，并从版本号中删除了 ".0"。开发人员仍然使用 1.6.0 作为内部编号。此版本是在 JSR 270 下开发的。

Java 6 在 2013 年 2 月达到了支持生命周期的结束阶段，此时所有公共更新，包括安全更新，都将停止。Oracle 在 2013 年 3 月和 4 月发布了 Java 6 的两个更新，修复了一些安全漏洞。

此版本的主要变化包括：

  • 支持较旧的 Win9x 版本的支持被取消；非官方地说，Java 6 更新 7 是在这些 Windows 版本上运行的最后一个 Java 版本。这被认为是由于更新 10 中的重大更改所导致的
  • 脚本语言支持（JSR 223）：用于与脚本语言紧密集成的通用 API，并内置了 Mozilla JavaScript Rhino 集成
  • 核心平台和 Swing 的显著性能改进
  • 通过 JAX-WS（JSR 224）提供了改进的 Web 服务支持
  • JDBC 4.0 支持（JSR 221）
  • Java 编译器 API（JSR 199）：允许 Java 程序选择并以编程方式调用 Java 编译器
  • 将 JAXB 升级到 2.0 版本：包括了 StAX 解析器的集成
  • 支持可插入注解（JSR 269）
  • 许多图形用户界面改进，如在 API 中集成 SwingWorker、表格排序和过滤、真正的 Swing 双缓冲（消除灰色区域效果）
  • JVM 改进包括：同步和编译器性能优化、新算法和对现有垃圾回收算法的升级，以及应用程序启动性能

# JavaSE 7（2011-07 发布）

Java SE 7 文档 (opens new window) ，API 文档 (opens new window)， Open JDK (opens new window)

代号为 Dolphin

JavaSE 7 的重要特性包括支持 try-with-resources、捕获多个异常等功能。

  • switch 语句中的 String（Java 7）
  • 二进制字面量（Java 7）
  • try-with-resources（Java 7）
  • 通过单个 catch 缓存多个异常（Java 7）
  • 数字字面量中的下划线（Java 7）

# JavaSE 8（LTS，2014-03 初版发布）

JDK 8 文档 (opens new window)，API 文档 (opens new window)，文档 (opens new window)，JDK 8 的新功能 (opens new window)，Open JDK (opens new window)

代号为 Spider。JavaSE 8 的重要特性包括 Lambda 表达式、方法引用、默认方法、函数式接口、Java 8 日期/时间、流类等。

# Java 编程语言

  • Lambda 表达式：通过消除匿名类的需求，大大简化了代码，并支持强大的函数式编程，可以内联定义函数式接口的实现
  • 方法引用：为具体已经有名称的方法提供了易读的 Lambda 表达式
  • 函数式接口：支持函数式编程的通用 API
  • 接口中的默认方法和静态方法
  • 类型推断：改进的类型推断
  • Optional 类以正确处理 null 值为目的，类似于 Guava 中的 Optional
  • 方法参数反射
  • 类型注解和可重复注解：类型注解允许在任何使用类型的地方应用注解，而不仅仅是在声明上。可重复注解允许将同一注解类型多次应用于同一声明或类型
  • 类型使用

# 集合类

  • 流式 API（java.util.stream）：新的流式 API 支持流水线处理。以声明式的方式处理数据，类似于 SQL。支持过滤、映射、限制、规约、查找、匹配、计数、排序、统计等聚合操作
  • forEach 方法
  • Collectors
  • StringJoiner
  • 并行数组排序
  • HashMap 在键冲突时的性能改进

# 脚本、日期时间、IO、安全等方面的改进

  • Base64 编码解码
  • Nashorn（替代了 Java 7 中的 Rhino）：用于执行 JavaScript 代码的基于 Java 的引擎。工具为 $JAVA_HOME/bin/jjs
  • Java 8 日期/时间 API
  • Java JDBC 改进：JDBC 4.2 中的新功能，例如 DriverAction，包括 Java DB 10.10 等
  • Java IO 改进，例如 - 阅读 (opens new window)
  • Java 并发改进 - 阅读 (opens new window)

# 工具

  • 新工具：添加了新的编译器工具和实用程序，如 jdeps 用于查找依赖关系
  • 提供了 jjs 命令来调用 Nashorn 引擎
  • 提供了 jarsigner 命令来调用 Nashorn 引擎

# Java 9（2017-09 发布）

JDK 9 文档 (opens new window)，API 文档 (opens new window)，JDK 9 的新功能 (opens new window)，Open JDK (opens new window)

Oracle 发布了 Java 9，引入了一系列丰富的新功能。它包括对 Java 编程、JVM、工具和库的各种升级。其目的是将 JDK 和 Java 标准版平台模块化，以便可以在小型计算设备上进行缩小。Java 9 添加了 90 多项增强功能，其中最重要的如下所示：

  • 平台模块系统（Project Jigsaw）：引入了一种新的 Java 编程组件，即模块（module），它是一组具有名称和自描述性的代码和数据。
  • 多版本 JAR：增强了 JAR 格式，使得多个特定于不同 Java 版本的类文件可以共存于一个存档中
  • 接口私有方法
  • 带资源的 try 语句：现在可以将 final 变量用作 try-with-resources 语句中的资源
  • 匿名类
  • @SafeVarargs 注解
  • 集合工厂方法：为 List、Set 和 Map 接口添加了新的静态工厂方法，用于创建这些集合的不可变实例
  • 进程 API 改进：改进了用于控制和管理操作系统进程的 API
  • 新的版本字符串方案
  • JShell：Java Shell（REPL）：将读取-求值-打印循环（REPL）功能添加到 Java 平台
  • 进程 API 改进
  • 控制面板
  • Stream API 改进：通过允许过滤传入的对象序列化数据流，增强了安全性和健壮性
  • 面向 Microsoft Windows 的安装程序增强等
  • HTTP 2 客户端：新的 HTTPClient API，支持 WebSockets 和 HTTP 2 流以及服务器推送功能
  • 改进的 JavaDocs：支持生成 HTML5 输出。提供搜索框以生成的 API 文档
  • 私有接口方法：增强接口，添加了私有方法和私有静态方法
  • 增强的 @deprecated 注解：重新设计 @deprecated 注解，提供有关 API 状态和预期处理的更多信息
  • 内部类的 Diamond 运算符：如果推断类型的参数类型可以表示，允许在匿名类中使用 Diamond 运算符
  • Optional 类的改进：向 java.util.Optional 类添加了一些有用的新方法
  • 多分辨率图像 API：支持将具有不同分辨率的一组图像封装到一个单一的多分辨率图像中
  • CompletableFuture API 改进：CompletableFuture 类的异步机制可以通过 ProcessHandle.onExit 方法在进程退出时执行操作
  • 轻量级 JSON：引入了轻量级的 API，可以在 Java 9 中使用 JSON 来消费和生成文档和数据流
  • 响应式流 API：Java SE 9 中引入了新的响应式流 API，支持响应式编程

# Java 10（2018-03 发布）

JDK 10 文档 (opens new window), API (opens new window),JDK 10 新特性 (opens new window),Open JDK (opens new window)

  • JEP 286：局部变量类型推断
  • 不可修改的集合：java.util.List、java.util.Map和java.util.Set 分别新增了一个静态方法 copyOf(Collection)。toUnmodifiable*() java.util.stream.Collectors 还新增了一些方法，用于将 Stream 收集成不可修改的 List、Map 或 Set
  • JEP 296：将 JDK 森林合并为一个单一仓库
  • JEP 304：垃圾收集器接口：通过引入干净的垃圾收集器（GC）接口，提高不同垃圾收集器的源代码隔离性
  • JEP 307：G1 的并行 Full GC：通过并行化完整的 GC，改善 G1 最坏情况下的延迟
  • JEP 310：应用类数据共享：该 JEP 扩展了现有的类数据共享（"CDS"）功能
  • JEP 312：线程本地握手
  • JEP 314：附加的 Unicode 语言标签扩展
  • JEP 316：在替代存储设备上进行堆分配：它使得 HotSpot VM 可以在用户指定的替代存储设备（如 NV-DIMM）上分配 Java 对象堆。
  • JEP 317：实验性的基于 Java 的 JIT 编译器：它将增强 java.util.Locale 和相关 API，以实现 BCP 47 语言标签的附加 Unicode 扩展。
  • JEP 319：根证书：在 JDK 中提供了一组默认的根证书颁发机构（CA）证书。
  • JEP 322：基于时间的版本命名
  • JEP 313：移除本地头文件生成工具（javah）
  • Java 10 中的已弃用特性和选项
