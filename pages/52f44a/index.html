    # Arthas简介

    在学习Arthas之前，推荐先看上一篇美团技术团队的Java 动态调试技术原理及实践 (opens new window)，这样你会对它最底层技术有个了解。可以看下文中最后有个对比图：Greys(Arthas也是基于它做的二次开发)和Java-debug-tool。

    # Arthas是什么

    Arthas 是Alibaba开源的Java诊断工具，深受开发者喜爱。

    # Arthas能解决什么问题

    当你遇到以下类似问题而束手无策时，Arthas可以帮助你解决：

    • 这个类从哪个 jar 包加载的? 为什么会报各种类相关的 Exception?
    • 我改的代码为什么没有执行到? 难道是我没 commit? 分支搞错了?
    • 遇到问题无法在线上 debug，难道只能通过加日志再重新发布吗?
    • 线上遇到某个用户的数据处理有问题，但线上同样无法 debug，线下无法重现！
    • 是否有一个全局视角来查看系统的运行状况?
    • 有什么办法可以监控到JVM的实时运行状态?

    Arthas支持JDK 6+，支持Linux/Mac/Windows，采用命令行交互模式，同时提供丰富的 Tab 自动补全功能，进一步方便进行问题的定位和诊断。

    # Arthas基于了哪些工具上发展而来

    • greys-anatomy 在新窗口打开 (opens new window): Arthas代码基于Greys二次开发而来，非常感谢Greys之前所有的工作，以及Greys原作者对Arthas提出的意见和建议！
    • termd 在新窗口打开 (opens new window): Arthas的命令行实现基于termd开发，是一款优秀的命令行程序开发框架，感谢termd提供了优秀的框架。
    • crash 在新窗口打开 (opens new window): Arthas的文本渲染功能基于crash中的文本渲染功能开发，可以从这里 在新窗口打开 (opens new window)看到源码，感谢crash在这方面所做的优秀工作。
    • cli 在新窗口打开 (opens new window): Arthas的命令行界面基于vert.x提供的cli库进行开发，感谢vert.x在这方面做的优秀工作。
    • compiler 在新窗口打开 (opens new window) Arthas里的内存编绎器代码来源
    • Apache Commons Net 在新窗口打开 (opens new window) Arthas里的Telnet Client代码来源
    • JavaAgent：运行在 main方法之前的拦截器，它内定的方法名叫 premain ，也就是说先执行 premain 方法然后再执行 main 方法
    • ASM：一个通用的Java字节码操作和分析框架。它可以用于修改现有的类或直接以二进制形式动态生成类。ASM提供了一些常见的字节码转换和分析算法，可以从它们构建定制的复杂转换和代码分析工具。ASM提供了与其他Java字节码框架类似的功能，但是主要关注性能。因为它被设计和实现得尽可能小和快，所以非常适合在动态系统中使用(当然也可以以静态方式使用，例如在编译器中)

    # Arthas 命令集

    # 基础命令

    • help——查看命令帮助信息
    • cat 在新窗口打开 (opens new window)——打印文件内容，和linux里的cat命令类似
    • [grep]](https://github.com/alibaba/arthas/blob/master/site/src/site/sphinx/grep.md)——匹配查找，和linux里的grep命令类似
    • pwd 在新窗口打开 (opens new window)——返回当前的工作目录，和linux命令类似
    • cls——清空当前屏幕区域
    • session——查看当前会话的信息
    • reset 在新窗口打开 (opens new window)——重置增强类，将被 Arthas 增强过的类全部还原，Arthas 服务端关闭时会重置所有增强过的类

    # monitor/watch/trace相关

    请注意，这些命令，都通过字节码增强技术来实现的，会在指定类的方法中插入一些切面来实现数据统计和观测，因此在线上、预发使用时，请尽量明确需要观测的类、方法以及条件，诊断结束要执行 shutdown 或将增强过的类执行 reset 命令。

    • monitor 在新窗口打开 (opens new window)——方法执行监控
    • watch 在新窗口打开 (opens new window)——方法执行数据观测
    • trace 在新窗口打开 (opens new window)——方法内部调用路径，并输出方法路径上的每个节点上耗时
    • stack 在新窗口打开 (opens new window)——输出当前方法被调用的调用路径
    • tt 在新窗口打开 (opens new window)——方法执行数据的时空隧道，记录下指定方法每次调用的入参和返回信息，并能对这些不同的时间下调用进行观测

    # 重要：如何临时更改代码运行?

    场景：我找到了问题所在，能否线上直接修改测试，而不需要在本地改了代码后，重新打包部署，然后重启观察效果?

    # 先反编译出class源码
jad --source-only com.example.demo.arthas.user.UserController > /tmp/UserController.java  

# 然后使用外部工具编辑内容
mc /tmp/UserController.java -d /tmp  # 再编译成class

# 最后，重新载入定义的类，就可以实时验证你的猜测了
redefine /tmp/com/example/demo/arthas/user/UserController.class
