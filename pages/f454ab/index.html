SpringCloud - API网关

# SpringCloud API网关

本章节介绍SpringCloud中的API网关组件，包括Zuul、Gateway和Sentinel Gateway。@pdai

# 一、Zuul

# 1.1 Zuul 简介

Zuul是Netflix开源的API网关，是SpringCloud Netflix套件中的核心组件之一。它提供了路由、过滤、负载均衡等功能，是微服务架构中的统一入口。

核心功能：

  • 路由：将请求路由到对应的服务
  • 过滤：对请求进行过滤和处理
  • 负载均衡：集成Ribbon，支持负载均衡
  • 熔断降级：集成Hystrix，支持熔断降级
  • 认证授权：对请求进行认证和授权

# 1.2 Zuul 核心功能

路由功能：

  • 简单路由：直接将请求路由到指定服务
  • 动态路由：根据配置动态路由请求
  • 正则路由：使用正则表达式匹配路由

过滤功能：

  • 前置过滤：在请求路由前执行
  • 路由过滤：在请求路由时执行
  • 后置过滤：在请求路由后执行
  • 错误过滤：在请求处理出错时执行

# 1.3 Zuul 路由配置

配置示例：

zuul:
  routes:
    user-service:
      path: /user/**
      serviceId: user-service
      stripPrefix: false
    order-service:
      path: /order/**
      serviceId: order-service
      stripPrefix: false
  prefix: /api  # 统一前缀
  ignored-services: '*'  # 忽略所有未明确配置的服务
  sensitive-headers: Cookie,Set-Cookie,Authorization  # 敏感头信息

# 1.4 Zuul 过滤器

自定义过滤器：

@Component
public class AuthenticationFilter extends ZuulFilter {
    @Override
    public String filterType() {
        return "pre"; // 前置过滤器
    }
    
    @Override
    public int filterOrder() {
        return 0; // 过滤器顺序
    }
    
    @Override
    public boolean shouldFilter() {
        return true; // 是否执行过滤器
    }
    
    @Override
    public Object run() throws ZuulException {
        RequestContext context = RequestContext.getCurrentContext();
        HttpServletRequest request = context.getRequest();
        
        // 认证逻辑
        String token = request.getHeader("Authorization");
        if (token == null || !token.startsWith("Bearer ")) {
            context.setSendZuulResponse(false);
            context.setResponseStatusCode(HttpStatus.UNAUTHORIZED.value());
            context.setResponseBody("Unauthorized");
        }
        
        return null;
    }
}

# 二、Gateway

# 2.1 Gateway 简介

Spring Cloud Gateway是Spring官方提供的API网关，是SpringCloud生态系统中的核心组件。它基于Spring 5、Spring Boot 2和Project Reactor，提供了更加现代和高效的API网关功能。

核心功能：

  • 路由：将请求路由到对应的服务
  • 过滤：对请求进行过滤和处理
  • 断言：根据条件匹配请求
  • 集成：与SpringCloud生态系统集成
  • 反应式：基于反应式编程模型

# 2.2 Gateway 核心功能

路由：

  • Path路由：基于路径匹配
  • Host路由：基于主机名匹配
  • Method路由：基于HTTP方法匹配
  • Query路由：基于查询参数匹配
  • Header路由：基于请求头匹配

过滤：

  • Pre过滤器：在请求路由前执行
  • Post过滤器：在请求路由后执行
  • Route过滤器：在特定路由上执行
  • Global过滤器：在所有路由上执行

断言：

  • After断言：在指定时间后匹配
  • Before断言：在指定时间前匹配
  • Between断言：在指定时间范围内匹配
  • Cookie断言：基于Cookie匹配
  • Header断言：基于请求头匹配
  • Host断言：基于主机名匹配
  • Method断言：基于HTTP方法匹配
  • Path断言：基于路径匹配
  • Query断言：基于查询参数匹配

# 2.3 Gateway 路由配置

配置文件：

spring:
  cloud:
    gateway:
      routes:
        - id: user-service
          uri: lb://user-service
          predicates:
            - Path=/user/**
          filters:
            - StripPrefix=1
        - id: order-service
          uri: lb://order-service
          predicates:
            - Path=/order/**
          filters:
            - StripPrefix=1
      globalcors:
        cors-configurations:
          '[/**]':
            allowed-origins: "*"
            allowed-methods: "*"
            allowed-headers: "*"

# 2.4 Gateway 过滤器

自定义过滤器：

@Component
public class AuthenticationFilter implements GlobalFilter, Ordered {
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();
        String token = request.getHeaders().getFirst("Authorization");
        
        if (token == null || !token.startsWith("Bearer ")) {
            exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);
            return exchange.getResponse().setComplete();
        }
        
        return chain.filter(exchange);
    }
    
    @Override
    public int getOrder() {
        return -100; // 过滤器顺序
    }
}

# 三、Sentinel Gateway

# 3.1 Sentinel Gateway 简介

Sentinel Gateway是Sentinel提供的API网关解决方案，它基于Spring Cloud Gateway，提供了流量控制、熔断降级、热点参数限流等功能。

核心功能：

  • 流量控制：基于QPS、并发数等指标进行流量控制
  • 熔断降级：当后端服务不可用时，自动触发熔断
  • 热点参数限流：对热点参数进行特殊处理
  • 系统保护：保护系统整体稳定性
  • 实时监控：提供实时监控和告警

# 3.2 Sentinel Gateway 流量控制

配置方式：

  1. 控制台配置：通过Sentinel控制台配置流量控制规则
  2. 代码配置：通过代码配置流量控制规则

代码示例：

// 初始化网关流量控制规则
private void initGatewayFlowRules() {
    Set<GatewayFlowRule> rules = new HashSet<>();
    GatewayFlowRule rule = new GatewayFlowRule();
    rule.setResource("user-service");
    rule.setCount(20); // QPS限制为20
    rule.setIntervalSec(1); // 统计时间窗口，单位为秒
    rules.add(rule);
    GatewayRuleManager.loadRules(rules);
}

配置方式：

// 初始化网关热点参数限流规则
private void initGatewayParamFlowRules() {
    Set<GatewayParamFlowRule> rules = new HashSet<>();

# 四、API网关的最佳实践

# 4.1 选择合适的API网关

  • Zuul：适合使用SpringCloud Netflix套件的项目
  • Gateway：适合使用SpringCloud官方推荐组件的项目，需要高性能
  • Sentinel Gateway：适合需要流量控制、熔断降级等功能的项目

# 4.2 路由配置最佳实践

  • 合理规划路由：根据业务需求规划路由规则
  • 使用服务名称：通过服务名称路由，避免硬编码IP地址
  • 设置合理的前缀：为不同服务设置合理的路径前缀
  • 配置CORS：合理配置CORS，支持跨域请求

# 4.3 过滤器使用最佳实践

  • 认证授权：在网关层统一处理认证和授权
  • 日志记录：在网关层统一记录请求日志
  • 流量监控：在网关层统一监控流量
  • 异常处理：在网关层统一处理异常

# 4.4 性能优化

  • 使用连接池：配置合理的连接池大小
  • 启用缓存：对频繁请求的资源启用缓存
  • 减少过滤链：只使用必要的过滤器
  • 合理配置线程池：根据实际情况配置线程池大小

# 4.5 高可用配置

  • 部署多个实例：部署多个网关实例
  • 使用负载均衡：在网关前面部署负载均衡器
  • 健康检查：配置健康检查，及时发现故障实例
  • 自动扩缩容：根据流量自动调整实例数量

# 五、小结

本章节介绍了SpringCloud中的API网关组件，包括：

  • Zuul的核心功能、配置和使用
  • Gateway的核心功能、配置和使用
  • Sentinel Gateway的核心功能、配置和使用
  • API网关的最佳实践

API网关是微服务架构中的重要组件，它作为系统的统一入口，负责路由、过滤、认证等功能。在实际项目中，应根据具体需求选择合适的API网关，并遵循最佳实践进行配置和使用。
